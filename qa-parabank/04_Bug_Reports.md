# 3. Severity Definitions

| Severity    | Description                                                                                        |
| ----------- | -------------------------------------------------------------------------------------------------- |
| 🔴 Critical | Prevents use of a core banking feature, causes data loss, or presents a significant security risk. |
| 🟠 High     | Major functionality is affected, but a workaround may exist.                                       |
| 🟡 Medium   | Feature works incorrectly but does not block core functionality.                                   |
| 🔵 Low      | Minor UI, cosmetic, or usability issue with little impact on functionality.                        |

# 6. Bug Report Template

Every defect should be documented in its own Markdown file using the following naming convention:

```
BUG001_Short_Description.md
BUG002_Short_Description.md
BUG003_Short_Description.md
```

Each bug report should include:

* Bug ID
* Title
* Environment
* Browser
* Browser Version
* Operating System
* Related Test Case
* Severity
* Priority
* Status
* Preconditions
* Steps to Reproduce
* Expected Result
* Actual Result
* Impact
* Attachments (Screenshots / Video)
* QA Assessment
* Test Evidence
