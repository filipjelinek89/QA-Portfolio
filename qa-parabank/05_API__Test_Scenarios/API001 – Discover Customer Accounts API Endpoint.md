# API001 – Discover Customer Accounts API Endpoint

**Module:** API Testing

**Priority:** Medium

**Severity:** N/A

**Type:** API Discovery

---

## Objective

Identify the API endpoint responsible for retrieving customer account information by inspecting network traffic generated by the ParaBank web application.

This test focuses on understanding how the frontend communicates with the backend before performing API requests directly in Postman.

---

## Preconditions

- ParaBank application is accessible.
- A valid user account is available.
- Google Chrome is installed.
- Chrome Developer Tools are available.

---

## Request Information

| Field | Value |
|------|------|
| Tool Used | Google Chrome Developer Tools |
| Network Filter | Fetch/XHR |
| HTTP Method | GET |
| Endpoint | `/parabank/services_proxy/bank/customers/{customerId}/accounts` |
| Authentication | Authenticated browser session (JSESSIONID) |

---

## Test Steps

| Step | Action |
|------|--------|
| 1 | Open the ParaBank application in Google Chrome. |
| 2 | Log in using valid user credentials. |
| 3 | Open Chrome Developer Tools (F12). |
| 4 | Select the **Network** tab. |
| 5 | Filter requests using **Fetch/XHR**. |
| 6 | Navigate to **Accounts Overview**. |
| 7 | Locate the request that retrieves customer account information. |
| 8 | Inspect the request method, endpoint, response status, and response payload. |

---

## Expected Result

- A GET request is generated when opening **Accounts Overview**.
- The endpoint responsible for retrieving customer accounts is identified.
- The request returns **HTTP 200 OK**.
- The response contains valid JSON with customer account information.

---

## Actual Result

The customer accounts endpoint was successfully identified using Chrome Developer Tools.

The application generated a **GET** request to:

`/parabank/services_proxy/bank/customers/{customerId}/accounts`

The request returned **HTTP 200 OK** and the response contained valid JSON with the customer's account information.

---

## Status

**PASS**

---

## Notes

This test was performed using **Chrome Developer Tools** and an authenticated browser session.

The purpose of this scenario is API discovery rather than API execution.

The identified endpoint will be used in the next test case (**API002**) to perform direct API validation using **Postman**.
