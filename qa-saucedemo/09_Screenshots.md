# Overview

This section contains screenshots collected during the manual testing of the SauceDemo web application.

Screenshots provide visual evidence of application behavior, support reported defects, and improve the overall quality and traceability of the testing documentation.

Whenever possible, screenshots were captured immediately after reproducing an issue to accurately reflect the application state.

---

# Purpose

The screenshots included in this portfolio serve the following purposes:

- Document failed test executions
- Provide visual evidence for reported defects
- Support defect analysis and troubleshooting
- Improve communication between testers and developers
- Enhance the readability of testing documentation

# Screenshot Categories

## рџђћ Bug Reports

This folder contains screenshots referenced in the individual bug reports.

Each screenshot captures the application state at the moment the defect was reproduced and provides visual evidence supporting the documented issue.

**Examples:**

- Login validation errors
- Shopping cart defects
- Checkout validation issues
- Navigation problems
- Accessibility observations
- User interface inconsistencies

# Usage Guidelines

To maintain consistency throughout the project:

- One screenshot should represent one issue whenever possible.
- File names should match the corresponding Bug ID.
- Images should clearly display the observed behavior.
- Sensitive or unnecessary browser information should be excluded.
- Screenshots should be captured at a readable resolution.

---

# Summary

The screenshot repository complements the manual testing documentation by providing visual evidence for reported defects and important application screens.

Organizing screenshots separately from the documentation keeps the repository structured, improves maintainability, and makes it easier for reviewers and recruiters to navigate the project.
